# RULE-0001

## Rule ID

RULE-0001

## Rule Statement

When reviewing or preparing IPTS-specific assets, use the approved IPTS reference pattern before treating any IPTS output structure as valid: identify the output mode taxonomy, check the relevant source type mapping, compare against reviewed examples for the selected mode, and treat full setup block labels as reference structure rather than automatic output instructions.

## Purpose

Ensure IPTS-specific assets are reviewed against approved knowledge before operational use.

## Scope

IPTS project-specific module rule.

## Exceptions

- Does not apply outside IPTS-specific assets.
- Does not authorize Core, Behavior, or Architecture changes.
- Does not authorize output generation without a separate approved workflow.
- Does not override verification, evidence, or human review requirements.

## Test or Observable Review Criteria

A review passes only when all of the following are recorded for the IPTS-specific asset:

1. The applicable output mode taxonomy is identified.
2. The relevant source type mapping is checked.
3. At least one reviewed example is compared for the selected mode.
4. Full setup block labels are treated as reference structure only, not automatic output instructions.
5. Missing evidence, conflicts, or unresolved uncertainty are escalated for human review.

A review fails when any required step is absent, when a setup block label is treated as an automatic instruction, or when the rule is used to authorize output generation.

## Required Review Record

Each application of this rule must produce a review record containing:

- Required Review Artifact: the IPTS-specific asset or review package under review, identified by project-relative path or stable reference.
- Output Mode Taxonomy: the selected mode and its source reference.
- Source Type Mapping: the mapping used and its source reference.
- Reviewed Example: at least one reviewed example identified by stable reference, with comparison notes.
- Setup Block Label Handling: explicit confirmation that labels were treated as reference structure only.
- Required Evidence for Pass: completed entries for all five pass criteria, source references for the taxonomy, mapping, and example, and confirmation that no output-generation authority was granted.
- Required Evidence for Fail: the failed criterion, observed condition, affected artifact, and supporting source or review reference.
- Escalation Record: issue type, description, evidence reference, reviewer action, disposition, and whether human review remains pending.
- Human Reviewer: named reviewer responsible for the review result.
- Review Date: date on which the review was completed.
- Final Review Status: Pass, Fail, or Needs Human Review.
