# Contributing

Keep changes small, traceable, and consistent with the approved Elias Constitution.

- Do not add secrets, personal data, browser data, backups, logs, ZIP archives, or unrelated project files.
- Do not modify Core policy automatically.
- Every operational rule proposal needs scope, evidence, justification, and observable test criteria.
- Run `npm test` before submitting changes.
- Clearly label mock, simulated, incomplete, and unverified behavior.
- Do not claim production readiness without executable evidence.
